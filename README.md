# Work Hance Assignment

## Project Overview
This assignment demonstrates my understanding of front-end fundamentals, including responsive design, UI structuring, and client-side validation.


## Part 1: Quick Questions

### 1. Role of HTML, CSS, and JavaScript
HTML is responsible for defining the structure and content of a webpage, such as headings, paragraphs, and forms.  
CSS is used to control the visual presentation, including layout, colors, spacing, and responsiveness.  
JavaScript adds interactivity and dynamic behavior, such as handling user input, validating forms, and updating content without reloading the page.

---

### 2. How do you make a website responsive?
To make a website responsive, I use flexible layout systems like Flexbox and CSS Grid so elements can adapt to different screen sizes.  
I apply media queries to adjust layouts for devices like tablets and mobiles.  
I also use relative units such as percentages, rem, and viewport units instead of fixed pixel values, and ensure images scale properly using max-width.

---

### 3. What steps would you take if a website is loading slowly?
I would first identify performance bottlenecks using browser developer tools.  
Then I would optimize images by compressing them and using appropriate formats.  
I would minimize and remove unused CSS and JavaScript, reduce unnecessary network requests, and implement lazy loading where needed.  
Additionally, I would use caching techniques and consider using a CDN to improve load times.

---

### 4. How do you handle form validation?
I use a combination of HTML and JavaScript for form validation.  
HTML provides basic validation through attributes like required and input types such as email.  
JavaScript is used for custom validation logic, such as checking empty fields, validating email formats, and displaying meaningful error messages.  
Form submission is prevented until all validations pass.

---

### 5. What are 2–3 key points you follow to design a clean UI?
I focus on maintaining consistent spacing and alignment to improve readability.  
I use a limited and consistent color palette along with clear typography to ensure visual clarity.  
I also keep the layout simple and avoid unnecessary elements so the user can focus on the main content.

---

## Part 2: Implementation

This project is a responsive landing page for Work Hance built using HTML, CSS, and JavaScript.

Key features include:
- A structured layout with header, hero, services, and contact sections
- Responsive design using Flexbox and media queries for mobile compatibility
- A contact form with validation to prevent empty or incorrect submissions
- Basic interactivity such as hover effects to enhance user experience
